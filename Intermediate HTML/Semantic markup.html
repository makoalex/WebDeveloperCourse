Semantic Markup

Enhances accessability, readability, searchability, interoperability. In other words using semantic markup is important if you want your site accessible to all visitors

We write good semantic markup by selecting HTML tags properly, tags that convey meaning about that information to be used

there are elements that are semantic, and non- semantic. Non- semantic:

  1. div
  2. span

main semantic tags

  • main- high level element used to contain all the elements of a single webpage
  • nav- contains blocks of the site navigation links. Placed in the header and footer or aside (sidebar)
  • header- used for webpage header, has the Logo site navigation,
  • footer used for a web page footer which typically contains authorship, contact, and copyright information in addition to navigational links
  • article- self contained composition in an article page application or site
  • section used to markup sections of a document, chapter
  • aside portion of a document whose content is indirectly related to the docs main content.

In the past, the div element was the main way sections of a website were identified and grouped

and they can be grouped into 4 semantic categories

  • document structure tags
  • textual meaning tags
  • media type tags
  • correlation tags
